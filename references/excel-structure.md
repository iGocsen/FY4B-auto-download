# Excel 结构参考

## 时间控制表（D:\云图\FY4B\待导入日期\卫星云图导入时间控制.xlsx）

### B 列公式链

- B2：`=B161+1/96`（每行 +15 分钟，96 行 = 24 小时）
- B3：`=B2+1/96`
- …依次递推…
- B161：`=B160+1/96`
- B162：手动填入的下一批次过期时间（由脚本更新）

### 更新流程

1. 脚本读取 B1（当前批次过期时间，北京时间）
2. 判断是否过期 → 如过期则更新 Excel
3. 更新逻辑：将 B162 的值写入 B2，触发公式链重算
4. 循环直到 B1 ≥ now - 12 小时（即剩余有效期 ≥ 12 小时则停止）

### 关键地址

| 地址 | 内容 | 用途 |
|------|------|------|
| B1 | 当前批次过期时间 | 判断是否需要更新 |
| B2 | `=B161+1/96` | 公式链起点，重写后触发重算 |
| B161 | 最后一行（倒数第二个时间） | 公式链终点 |
| B162 | 下一批次过期时间（手动填入值） | 复制到 B2 的数据源 |

### 时区注意

Excel COM 返回的 datetime 带有本地时区信息（offset-aware），而 `datetime.now()` 为 offset-naive。比较前需统一处理：
```python
def ole_to_datetime(val):
    if isinstance(val, datetime):
        return val.replace(tzinfo=None)  # 去除时区，转为 naive
    return val
```

### 过期判断条件

- 旧逻辑：B1 ≥ 当前时间（追到未来，可能循环几十次）
- 当前逻辑：B1 ≥ 当前时间 - 12 小时（只要还有 12 小时有效期就停止）
